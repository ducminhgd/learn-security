# Output rules

1. There must be a `SYLLABUS.md` file to show the structure of the course.
2. There are max 3 levels of directories for subjects, for topics that I will learn.
3. Each day will be markdown file for the knowledge that enough for one day learning. Format `DAY-<DAY SEQUENCE>-<Title>.md`. `DAY SEQUENCE` is 4 digits with zero paddings as the sequence of days.
4. A topic or a subject can be divided into multiple days.
5. A lesson can be divided into multiple episodes.
6. A lesson or an episode should have: Goals, prerequisited lessons or knowledge (if any), main content, key take-aways, and/or exercises if needed.
7. The should be a place of the file, where I can ask for the questions.
   1. If I asks the questions, you can write the questions into that part, and write the answers, follow-up questions, too.
   2. IF I notes the questions into the file, I can ask you to answer them.
   3. Each question has an Global ID, and has a hierachy with follow up questions.
   4. Use numbering for bulletings.
8. Use front-matter to note the title, the tags, the related topics, or fields.
9. Example codes can be generated in `samples`, and linked from the related lessons.
10. Preqrequisted knowledge can be generated in `knowledge-base` as markdown files.
